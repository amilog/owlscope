## Is OwlScope free?

Yes. MIT licensed. Source on [GitHub](https://github.com/amilog/owlscope).

## Does my data leave my machine?

No. The desktop app runs a WebSocket server on `localhost`. Your SDK
connects to it directly. Nothing is uploaded anywhere.

## Does it work in production?

The SDK is **no-op in production builds** (`NODE_ENV=production` for
React Native, `kReleaseMode` for Flutter). Even if shipped, it does
nothing.

## Does it work on iOS device / Android device?

Yes — that's why both SDKs ship a `setup` command:

- **React Native**: `npx owlscope setup` patches `Info.plist` and runs
  `adb reverse` for connected devices.
- **Flutter**: `dart run owlscope:setup` patches native configs;
  `dart run owlscope:reverse` handles Android USB.

After the one-time setup, all four targets (sim/device, both platforms)
work without further tweaking.

## My port 9090 is taken.

Pass `port: 9091` to both `startOwlScope()` / `owlscopeAuto()` and
launch the desktop with the same port. Or stop the conflicting service.

## Can I write my own UI?

Yes. The wire protocol is documented in
[`@owlscope/protocol`](https://github.com/amilog/owlscope/tree/main/packages/protocol).
The desktop app is just one consumer.

## Where do I report bugs?

[GitHub Issues](https://github.com/amilog/owlscope/issues).
