OwlScope has two pieces:

1. **The desktop app** — receives events, draws charts
2. **The SDK** — runs in your mobile app and streams events

Install both, run one setup command, write one line.

## 1. Desktop app

Download the latest build from the
[releases page](https://github.com/amilog/owlscope/releases), or build
from source:

```sh
git clone https://github.com/amilog/owlscope
cd owlscope
npm install
npm run dev
```

The desktop binds a WebSocket server on `0.0.0.0:9090`. Allow incoming
connections if macOS prompts.

## 2. SDK

<Tabs syncKey="platform">
  <TabItem label="React Native">
    ```sh
    npm install owlscope
    npx owlscope setup
    ```

    ```ts title="index.js — top of file"
    import { startOwlScope } from 'owlscope/rn';
    if (__DEV__) startOwlScope();

    import { AppRegistry } from 'react-native';
    import App from './App';
    import { name as appName } from './app.json';
    AppRegistry.registerComponent(appName, () => App);
    ```

    `npx owlscope setup` patches your `ios/<App>/Info.plist` for iOS 14+
    local-network permissions and runs `adb reverse` for connected
    Android devices. Idempotent — re-run any time.
  </TabItem>
  <TabItem label="Flutter">
    ```yaml title="pubspec.yaml"
    dependencies:
      owlscope: ^0.1.3
    ```

    ```sh
    flutter pub get
    dart run owlscope:setup
    ```

    ```dart title="lib/main.dart"
    import 'package:owlscope/auto.dart';
    import 'package:flutter/material.dart';

    void main() => owlscopeAuto(() => runApp(const MyApp()));
    ```

    For physical Android via USB:

    ```sh
    dart run owlscope:reverse
    ```
  </TabItem>
</Tabs>

## 3. Run

```sh
# Terminal 1
npm run dev   # at the OwlScope repo root

# Terminal 2
npm run ios    # or run-android, or flutter run, or yarn ios…
```

Your app appears in the desktop sidebar within a couple of seconds.
Click around, send a request, watch logs and network traffic stream in.
